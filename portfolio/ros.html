ROS Kinetic for Slackware Linux

Overview

I started this project at the time I first needed to use ROS. Being a firm supporter of Slackware Linux, I did not want to use VM since it can be very uncomfortable. Thus I decided to bring support for ROS on Slackware Linux. Currently, I am the main maintainer for ROS on Slackware. You can check the guide on the ROS wiki and my GitHub repo with additional packages.

The robot is first of its kind. I am currently writing an academic paper on the project.

rospkg, rosdep and rosdistro for Slackware

ROS has these internal tools that it uses to determine your OS distro in order to check for and install dependencies automatically. They are all implemented in python, so I just needed to write implementation of these tools for Slackware. The hardest part was finding a proper documentation as at some point I had read through a large part of the code to understand how it works.

The next part was less fun as I needed to define all Slackware software packages that ROS depends on. This required some more time and research but I finally got there. It was kind of interesting since I learned more about the packages that exist out there.

SlackBuilds

Unfortunately some of the dependencies did not have Slackware packages, so I had to create them. Of course one can always install from source, but this is not always robust and is not the Slackware way. I ended up creating Slackware packages for pcl, gazebo, ignition-math2, ignition-transport, flann, console_bridge, urdfdom. It was not straightforward as I ran into some runtime errors due to incorrect configuration, linking problems and writing patches. It was fun as I made contribution to Slackware too and got much better at cmake and scripting.

Building ROS

Lastly I had to build ROS. This was not straightforward either since Slackware manages its OS packages differently and this required its fair share of little cmkae patches to fix linking errors. I also got to discover a major bug on x86_64 architectures in the ROS build system that is currently beign fixed
