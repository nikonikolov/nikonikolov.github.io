Drone Delivery

Overview

The goal of this project is to build a drone that is able to deliver food around Brown University in Providence, RI. I am working together with Aaron Zhang - a student at Brown. He is responsible for the mechanics of the delivery mechanism, why I am developing the electronic hardware and the control algorithms for the drone. We are using a DJI M-100 drone. The project is partially sponsored by DJI.

Stay tuned for our first demo - coming very soon.

First Prototype

The type of control that DJI M-100 provides is accomplished by serial connection and a specific protocol implemented by DJI. It allows for an onboard embedded system to connect to the autopilot and send commands such as waypoint navigation.

My choice of onboard embedded system was a Raspberry Pi 3 since the DJI SDK currently requires a thread-safe operating system to run on and linux seems like the best choice. For our first prototype we are going to use the Pi to partially navigate the drone using the SDK and on a given signal by the remote control of the DJI, activate a mechanism that drops the package. We are planning to automate this in the future, but for now we are going for the "minimum viable product" concept.

Our delivery mechanism is spool based and is managed by servo motors. Luckily, my libdnx library (the same one I wrote for the Walking Quad) supports Raspberry Pi so I chose the same servos. I also had to simplify the electronics system as much as possible as my friend is setting everything up and it is really hard to make everything work from distance.
